## 2. P0 人工验收

### AI-101 首次进入

1. 启动应用并进入“AI 驾驶舱”。
2. 页面默认展示今日视图。
3. 无持仓时展示空状态；有持仓时展示总市值、今日变化、配置和影响项。
4. 页面不得因默认 Ollama 未启动而崩溃。

### AI-102 快照刷新

1. 记录证据栏快照编号。
2. 普通切换页面后五分钟内应复用编号。
3. 点击刷新后应生成新编号。
4. 旧快照仍可在 SQLite 中按 ID 读取。

### AI-103 数据缺口

1. 准备有成本金额但没有份额的持仓。
2. 断开一个行情源后刷新。
3. 资产仍出现在组合中，估值回退到成本金额。
4. 证据栏显示缺少份额或行情，涨跌覆盖率低于 100%。

### AI-104 今日变化口径

1. 使用当前市值 110、涨幅 10% 的固定资产。
2. 今日变化应为 10，而不是 11。
3. 组合涨跌率分母只包含有涨跌数据的资产。

### AI-105 组合视图

1. 切换“组合”。
2. 最大单项、前五项、行情完整度与表格数值一致。
3. 最大单项达到 30% 时显示集中度提醒。
4. 表格按当前市值降序且多分组同代码不产生重复 React key。

### AI-106 固定快照对话

1. 用快照 S1 发送问题。
2. 生成过程中强制刷新得到 S2。
3. 回答的上下文事件、模型上下文和保存消息仍引用 S1。
4. 模型不得重新计算或覆盖 Rust 指标。

### AI-107 模型设置

1. 分别选择 OpenAI、Claude、兼容接口。
2. 保存配置后不重启立即发送问题。
3. 同提供商已保存 Key 时，留空测试仍可使用旧 Key。
4. 切换提供商且未填新 Key 时，云提供商配置应校验失败。
5. GET 配置响应不得包含 Key 明文。

### AI-108 Ollama

1. 启动本地 OpenAI 兼容服务或 Ollama。
2. 地址使用 `http://127.0.0.1:11434/v1`，Key 留空。
3. 连接测试和中文流式回复均成功。

### AI-109 停止生成

1. 使用慢速流模型发起长回答。
2. 点击停止按钮。
3. 前端立即结束生成状态，已接收文字保留，并可继续发问。

### AI-110 模型失败降级

1. 将模型地址设置为不可达端口。
2. 今日视图和组合视图仍正常。
3. 发问后返回“AI 解读暂不可用”的本地快照摘要。
4. 摘要与当前固定快照一致并保存到会话。

### AI-111 云模型脱敏审计

1. 使用本地 mock 伪装为非本机云地址并记录请求。
2. 请求中不得出现精确总金额、持仓金额或成本金额。
3. 请求可包含占比、涨跌幅、贡献率、估值依据和数据时间。
4. `ai_request_audits` 记录提供商、模式、快照和字段清单，不记录 Key 与正文。

### AI-112 会话恢复

1. 完成一轮问答后重启应用。
2. 最近会话按顺序恢复用户消息和模型回复。
3. 保存失败的回答显示未保存状态。

### AI-113 桌面布局

1. 在常用桌面窗口和窄窗口检查今日、组合、证据栏和设置弹窗。
2. 页面无横向溢出、文字遮挡、按钮位移或嵌套卡片。
3. 证据栏可收起，窄窗口下布局顺序清晰。
